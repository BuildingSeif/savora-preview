# SAVORA V9: CINEMATIC TRANSITIONS & VARIABLE CARD LAYOUT

## PART 1: CINEMATIC BROWSE-TO-COOK TRANSITION

When the user taps "Cuisiner" from the recipe detail, the transition to cook mode should feel like "lights dimming in a theater."

CSS additions:
```css
/* Cook mode entry overlay */
.cook-entry-overlay{
  position:fixed;
  inset:0;
  background:rgba(26,23,20,.0);
  z-index:500;
  pointer-events:none;
  transition:background .5s cubic-bezier(.16,1,.3,1);
}
.cook-entry-overlay.active{
  background:rgba(26,23,20,.95);
}
```

JavaScript: When go(4) is called (enter cook mode), add a brief overlay animation:
```javascript
// Override the cook mode entry in the existing go() function
// Before transitioning to S4, flash a dark overlay for 300ms
function enterCookMode(){
  var overlay = document.getElementById('cookEntryOverlay');
  if(!overlay){
    overlay = document.createElement('div');
    overlay.id = 'cookEntryOverlay';
    overlay.className = 'cook-entry-overlay';
    document.querySelector('.ph').appendChild(overlay);
  }
  overlay.classList.add('active');
  setTimeout(function(){
    // The actual go(4) happens here after the overlay
    overlay.classList.remove('active');
  }, 600);
}
```

## IMPLEMENTATION NOTES

1. The cook entry overlay uses a cubic-bezier easing curve (.16,1,.3,1) for a smooth, natural dimming effect
2. Variable card heights (3/4, 4/5 pattern) create visual rhythm and encourage scrolling
3. Hero card (16/9 aspect ratio) can be inserted at strategic points for featured recipes
4. Whitespace increases reduce cognitive load and improve visual hierarchy
5. Frosted glass effect on tab bar uses backdrop-filter with saturate boost for refinement
6. All dark theme variants included for accessibility
